- Claim 8 under-specifies inherited custom default response classes. It says the fast path is allowed only when the route's `response_class` argument is a `DefaultPlaceholder`, and its test examples cover explicit `response_class=...` cases, but it does not explicitly reject or require tests for `FastAPI(default_response_class=...)`, `APIRouter(default_response_class=...)`, or `include_router(..., default_response_class=...)` with a response model. In `fastapi/routing.py`, defaults are resolved through `get_value_or_default(...)` before `get_request_handler(...)` receives `response_class`; inherited custom defaults arrive there as concrete response classes, not placeholders. A refactor that interprets "no custom response class was set" at the route-decorator level could incorrectly take the raw `Response(media_type="application/json")` fast path and bypass custom default response rendering/media types.
