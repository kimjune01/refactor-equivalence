## Finding F1 — Shared schema helper loses the no-injection edge
**Severity**: warning
**Claim**: C2
**What**: The claim says to add one helper that "returns the sole property name from an object schema, when one exists" and use it from both `mapParams` and `DelegateInvocation.withUserHints`, but those two call sites currently have different fallback behavior. `mapParams` falls back to `{ prompt }` when the schema is not a record or has no single property. `withUserHints` only falls back to `prompt` when `properties` is a record with zero or multiple keys; if the schema itself is not a record, or `properties` is missing/non-record, it returns the original args without injecting hints. A helper that only returns `string | undefined` makes it ambiguous whether `undefined` means "use prompt" or "do not inject hints", so an implementer must guess and may accidentally inject hints into malformed/non-object remote schemas.
**Evidence**: `packages/core/src/agents/agent-tool.ts:106` returns `{ prompt }` for non-record schemas and for schemas without exactly one property; `packages/core/src/agents/agent-tool.ts:227` through `packages/core/src/agents/agent-tool.ts:248` returns `agentArgs` without injection when the schema or `properties` is not a record, but uses `prompt` only when `properties` is a record and does not have exactly one key.
**Fix**: Clarify C2 so the shared helper either preserves two distinct outcomes for `withUserHints` (`single property`, `prompt fallback`, `no hint injection`) or narrow the claim to share only the `isRecord(properties)`/`Object.keys(properties)` extraction while explicitly keeping `withUserHints`' non-record/missing-`properties` no-op behavior.
