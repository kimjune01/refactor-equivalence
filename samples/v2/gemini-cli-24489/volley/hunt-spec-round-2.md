## Finding F1 — C1 would drop required DelegateInvocation defaults if followed literally
**Severity**: blocker
**Claim**: C1
**What**: The claim says to rename `_toolName`/`_toolDisplayName` and pass the renamed parameters "through unchanged to `BaseToolInvocation`." In the current code, `DelegateInvocation` must not pass those optional values unchanged: it supplies fallbacks of `AGENT_TOOL_NAME` and `Invoke ${definition.displayName ?? definition.name}` before calling `super`. Those defaults are observable when tests call `AgentTool.createInvocation` directly without override names. Following the claim literally would leave `this._toolName`/`this._toolDisplayName` undefined, and the browser delegation path would pass undefined values into `BrowserAgentInvocation`, causing it to fall back to `browser_agent` / `Browser Agent` instead of the asserted unified `invoke_agent` identity.
**Evidence**: `packages/core/src/agents/agent-tool.ts:134` has optional constructor parameters, while `packages/core/src/agents/agent-tool.ts:140` and `packages/core/src/agents/agent-tool.ts:141` apply the required defaults. `packages/core/src/agents/agent-tool.ts:158` and `packages/core/src/agents/agent-tool.ts:159` pass those resolved values to the browser invocation. `packages/core/src/agents/agent-tool.test.ts:136` asserts `BrowserAgentInvocation` is called with `invoke_agent`, and `packages/core/src/agents/agent-tool.test.ts:141` asserts `Invoke Browser Agent`.
**Fix**: Narrow C1 to a pure parameter rename while explicitly preserving the existing fallback expressions in the `DelegateInvocation` `super(...)` call; do not require passing optional override parameters unchanged to `BaseToolInvocation`.
